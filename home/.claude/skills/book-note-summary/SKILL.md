# 読書メモ要約スキル

## 目的

ユーザーが読書中にメモした断片的なテキスト（気になった文章・感想・思考）を受け取り、後で内容を鮮やかに思い起こせる、255文字程度の示唆的なまとめを生成する。

## 入力形式

- **本のタイトル**（著者名があれば尚可）
- **読書メモ**：気になった文章の抜き書き、感想、疑問、思ったことなど、形式は問わない

## 出力仕様

| 項目 | 内容 |
|------|------|
| 文字数 | 255文字前後（±20文字程度は許容） |
| 文体 | だである調（ですます調は厳禁） |
| 目的 | 後から読んだとき、本の核心と自分の思考が蘇るような示唆的な文章 |
| 形式 | 連続した文章（箇条書き・見出し・改行区切りは使わない） |

## まとめ方の原則

1. **本の核心を一文で捉える**：著者が最も伝えたかったことは何か
2. **メモから示唆を引き出す**：ユーザーが気になった箇所の「意味」を抽象化する
3. **記憶の引き金を仕込む**：後で読んだとき「あ、そういう本だった」と蘇るような具体的なイメージや問いを含める
4. **自分ごとにする**：メモの中にユーザーの視点・感想があれば、それを活かして主観を織り込む

## 禁止事項

- ですます調の使用
- 箇条書き・番号リスト・見出し
- 「この本は〜について書かれています」のような無味乾燥な説明調
- 「本書の核心は」「この本が伝えたいのは」「〜というのがこの本の核心だ」のようなメタな宣言で要約を始めること（核心は直接言わず、文章から立ち上がらせる）
- メモの内容をただ並べるだけの羅列
- 300文字を大幅に超えること

## 出力例

**入力例：**
タイトル：『思考の整理学』外山滋比古
メモ：「グライダーと飛行機の違い」「自力で飛べない人間が増えている」「朝の頭は夜と違う、朝型思考を勧める」「忘れることも大事、一度忘れて蒸留されたものが本物」

**出力例：**
受動的に知識を積み上げるグライダー型の学びから脱却し、自ら問いを立てて飛ぶ飛行機型の思考者になれ。朝の頭で考えること、一度忘れてから戻ってくるアイデアこそ本物という逆説は、効率主義に慣れた脳に小気味よく刺さる。知的生産とは詰め込むことではなく、発酵させることだと気づかせてくれる一冊。（136文字）

## 処理手順

1. タイトルとメモを受け取る
2. メモから「著者の主張」「ユーザーが反応したポイント」「本の雰囲気・文脈」を読み取る
3. 上記の原則に沿って255文字前後のまとめを生成する
4. 文字数を確認し、大幅にずれていれば調整する
5. まとめのみを出力する（前置き・説明・文字数表示は不要）
